# MS Teams Outlook Notifier Bot

Этот скрипт предназначен для безопасного мониторинга корпоративного почтового ящика (через Microsoft Graph API) на наличие новых тикетов из ServiceNow (INC, RITM, SCTASK) и алертов сетевого оборудования (SolarWinds). 

Бот автоматически парсит системные письма, убирает лишний HTML-код, предотвращает дублирование сообщений и пересылает самую важную информацию напрямую в ваш канал или чат **Microsoft Teams** при помощи вебхуков. Данные не покидают корпоративный контур, что обеспечивает полное соблюдение политик ИБ.

## 🚀 Основные возможности

* **Умный парсинг ServiceNow:** Распознает инциденты (`INC`), запросы (`RITM`) и задачи каталога (`SCTASK`). Генерирует прямые кликабельные ссылки на портал.
* **SLA Алерты:** Распознает письма о нарушении SLA (`has reached ... SLA`) и автоматически переводит их в разряд критических.
* **Мониторинг SolarWinds:** Извлекает IP, локацию и статус оборудования. Автоматически помечает алерт нужным эмодзи (🔴 Down, 🟢 Up). Упавшее оборудование помечается как критический инцидент.
* **Визуальная цветовая дифференциация в Teams:** Критические инциденты, нарушения SLA и упавшие серверы приходят с красной маркировкой, стандартные запросы — с синей.
* **Фильтрация получателей:** Обрабатывает только те письма, которые направлены на определенные настроенные ящики рассылок (например, локальные саппорты стран: Kyrgyzstan, Kazakhstan, Uzbekistan и т.д.).
* **Игнорирование мусора:** Бот автоматически игнорирует Work Orders (`WO...`), письма о закрытии тикетов (`has been resolved/closed`) и отсеивает дублирующиеся уведомления об одном и том же тикете.

## 🛠 Установка и настройка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/schatten08/email_notifier_bot.git
   cd email_notifier_bot
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корне проекта и добавьте следующие переменные:
   ```env
   CLIENT_ID=ваш_azure_client_id
   CLIENT_SECRET=ваш_azure_client_secret
   TENANT_ID=ваш_azure_tenant_id
   TEAMS_WEBHOOK_URL=https://ваша_длинная_ссылка_на_вебхук_в_teams
   TARGET_EMAIL=mail1@epam.com, mail2@epam.com
   ```

4. **Первый запуск и авторизация:**
   ```bash
   python bot.py
   ```
   При первом запуске бот выдаст в консоли ссылку (для авторизации по протоколу Oauth2 Native Client). 
   1. Скопируйте её, откройте в браузере, авторизуйтесь под нужным корпоративным аккаунтом.
   2. Согласитесь на предоставление прав.
   3. Вас перекинет на пустую страницу. **Скопируйте полную ссылку из адресной строки браузера**, вставьте её в терминал и нажмите Enter. 
   4. Бот сохранит токен в файл `o365_token.txt` — после этого он сможет работать автономно.

## ☁️ Запуск на сервере (например, PythonAnywhere или локальном VPS)

1. Загрузите файлы `bot.py`, `requirements.txt`, `.env` и уже авторизованный файл **`o365_token.txt`** в папку на сервере.
2. Установите библиотеки: `pip3 install --user -r requirements.txt`
3. Запустите скрипт: `python3 bot.py` (или настройте как службу через systemd/pm2).
