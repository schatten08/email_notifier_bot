# Outlook Telegram Notifier Bot

Этот бот предназначен для мониторинга корпоративного почтового ящика (через Microsoft Graph API) на наличие новых тикетов из ServiceNow (INC, RITM, SCTASK) и алертов от SolarWinds. Бот автоматически парсит эти письма, убирает лишний HTML-код, извлекает самую важную информацию и отправляет аккуратные уведомления в Telegram.

## 🚀 Основные возможности

* **Умный парсинг ServiceNow:** Распознает инциденты (`INC`), запросы (`RITM`) и задачи (`SCTASK`).
* **Мониторинг SolarWinds:** Извлекает IP, локацию и статус оборудования. Автоматически помечает алерт нужным эмодзи (🔴 Down, 🟢 Up).
* **Фильтрация получателей:** Извлекает только те письма, которые направлены на определенные ящики рассылок (настраивается через `TARGET_EMAIL`).
* **Игнорирование мусора:** Бот автоматически игнорирует Work Orders (`WO...`), письма о закрытии тикетов (`has been resolved/closed`) и фильтрует дубликаты.
* **🔕 Тихий режим (Ночные смены):** С 22:00 до 08:00 бот присылает уведомления без звука, чтобы никого не будить. *Исключение: Priority 1 / Critical тикеты и упавшие серверы от SolarWinds — они всегда приходят со звуком.*
* **📊 Статистика:** Поддерживает команду `/status` в Telegram для проверки аптайма и количества обработанных писем.

## 🛠 Установка и настройка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/schatten08/email_notifier_bot.git
   cd email_notifier_bot
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корне проекта и добавьте следующие переменные:
   ```env
   CLIENT_ID=ваш_azure_client_id
   CLIENT_SECRET=ваш_azure_client_secret
   TENANT_ID=ваш_azure_tenant_id
   TELEGRAM_TOKEN=ваш_telegram_bot_token
   TELEGRAM_CHAT_ID=ваш_telegram_chat_id
   TARGET_EMAIL=mail1@example.com, mail2@example.com
   ```

4. **Первый запуск и авторизация:**
   ```bash
   python bot.py
   ```
   При первом запуске бот выдаст в консоли ссылку. Скопируйте её, откройте в браузере, авторизуйтесь под нужным аккаунтом Microsoft и дайте разрешение приложению. 
   После успешного входа вас перекинет на белую страницу (или выдаст ошибку браузера). **Скопируйте полную ссылку из адресной строки браузера**, вставьте её в терминал и нажмите Enter. 
   Бот сохранит токен в файл `o365_token.txt` и больше просить авторизацию не будет.

## ☁️ Запуск на сервере (например, PythonAnywhere)

1. Загрузите файлы `bot.py`, `requirements.txt`, `.env` и уже созданный файл **`o365_token.txt`** в папку на сервере. (Токен обязателен, чтобы не проходить авторизацию заново!).
2. Установите библиотеки: `pip3 install --user -r requirements.txt`
3. Запустите скрипт: `python3 bot.py`
